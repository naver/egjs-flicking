SSR 상황 등에서 초기 렌더링을 최적화할 수 있게 하는 Flicking 반응형 API 옵션<br />이 옵션들을 통해 Flicking 인스턴스가 완전히 초기화되기 전에 초기 상태 값을 설정할 수 있어,<br />실제 Flicking 상태가 적용될 때 불필요한 리렌더링을 방지할 수 있습니다.

**Type**: TSInterface

|PROPERTY|TYPE|DEFAULT|DESCRIPTION|
|:---:|:---:|:---:|:---:|
|defaultIndex|number|0|시작할 초기 패널 인덱스. currentPanelIndex와 indexProgress의 초기값을 설정합니다.<br />또한 isReachStart, isReachEnd 초기값 계산에도 영향을 줍니다.<br />|
|totalPanelCount|number|0|Flicking 인스턴스의 전체 패널 개수. totalPanelCount의 초기값을 설정하며<br />SSR 하이드레이션 과정에서 레이아웃 시프트를 방지하는 데 도움이 됩니다.<br />|

```js
const options = {
  defaultIndex: 2,
  totalPanelCount: 5
};
const reactiveObj = connectFlickingReactiveAPI(flicking, options);
```
